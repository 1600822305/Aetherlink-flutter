# 智能体模式 UI 设计稿（移动端）

> 状态：设计稿，待评审。配套《智能体模式-设计初稿》（架构/领域/引擎）。
> 范围：**只做移动端 UI**。桌面端将来完整重写，本稿只负责"留底"——
> 领域层 + 执行引擎 + 事件流 provider 全部纯 Dart（`features/agent/application/`），
> UI 全部收在 `presentation/mobile/`，桌面端届时直接消费同一套 provider 重画界面。

---

## 一、设计原则

1. **不是聊天界面**。聊天是"气泡对话"；智能体是"任务执行监控台"。
   信息主体是**时间线事件流 + 计划 + 工作台**，不是左右气泡。
2. **一屏聚焦，横滑扩展**。复用工作区已经验证过的「全屏横滑壳」交互
   （`PageView`，工作区页同款），agent 用 2 页：左事件流、右工作台，
   把不同关注面拆开而不是塞进一页滚动。
3. **UI 只渲染事件流**。页面所有内容都由 `AgentEvent` 序列 + 任务状态派生，
   UI 不持有业务状态 → 杀进程恢复、桌面端重写都"免费"。
4. **视觉语言沿用现有 App**：Material3、`_OutlinedCard` 式卡片、Lucide 图标、
   `CustomSwitch`、`AppToast`，不引入新设计系统。

## 二、信息架构：智能体 / 话题 两级（对齐聊天的 助手 / 话题）

已拍板：智能体模式和聊天一样有**侧边栏**，两级结构——

- **智能体（档案）**：可配置的专长角色，类似聊天里的助手。
  每个档案 = 名称/图标 + 系统提示 + **可用工具集勾选**（文件/终端/网搜/
  知识库/技能）+ **绑定工作区** + 权限偏好。由此实现不同专长的智能体：
  「编程」（文件+终端全开）、「网络研究」（网搜+知识库为主）、
  「PPT/文档」（文件写作+模板技能）……内置若干预设，用户可自建/编辑。
- **话题（任务）**：某个智能体下的一次任务执行 = 一条事件流
  （即架构稿的 `AgentTask`）。已拍板：工作区绑在智能体上（一个工作区
  对应一个智能体，在智能体设置里改），话题创建时直接继承。

```
聊天侧栏底部（与「工作区」「翻译」同排）─┐
                                        ▼
/agent ────── 智能体主界面 = 当前话题的任务工作台（两页横滑）
   │            顶栏「回聊天」⇄ 聊天主界面（模式互切，写 app_main_mode 持久化）
   ├─ 侧边栏（左滑出，SidebarHost 同款交互）：
   │      智能体列表（当前高亮）→ 选中后显示其话题列表 + 新建话题
   │      底部：新建/管理智能体、回聊天入口（与聊天侧栏对称）
   └─ 智能体编辑页：档案配置（提示词/工具集/绑定工作区/权限）
```

冷启动：读 `app_main_mode`（`chat` | `agent`）决定初始主界面；
agent 模式重启 → 恢复上次所在的智能体与话题（各自持久化 last-active）。

## 三、侧边栏（智能体 / 话题）

交互与聊天侧栏同款（`SidebarHost` 左滑出、非 Drawer）。tab 结构对齐聊天：
「智能体 / 话题 / 设置」三个分段 tab；设置 tab 是**智能体独立的**设置页
（不共享聊天代码，只是功能类型对齐——当前智能体信息、执行设置、事件流
显示、权限与审批）。tab 记忆与聊天同款：会话内内存态记住上次所在 tab，
重启回默认智能体 tab。

```
┌───────────────┬────────────────┐
│ 智能体          │  ← 上：智能体列表（横向头像条或纵向列表，当前高亮）
│ ◉ 编程         │
│ ○ 网络研究      │
│ ○ PPT 助手     │
│ ＋ 新建智能体   │  → 智能体编辑页
├───────────────┤
│ 话题（编程）     │  ← 下：当前智能体的话题列表
│ ▶ 进行中 (1)   │
│ ┌───────────┐ │  话题卡：
│ │● 修复登录崩溃│ │  · 左侧状态色点（呼吸动效=running）
│ │ aetherlink… │ │  · 标题（首条指令生成，可改）
│ │ 第12轮·6m ⚠│ │  · 工作区 chip + 状态徽标 + 最近事件摘要
│ └───────────┘ │  · waitingApproval/Input：高亮描边 + ⚠
│ 历史           │  ← done/failed/cancelled，倒序，可滑动删除
│ ✓ 升级依赖过CI │
│ ＋ 新建话题     │  → 干净空态草稿（见下）
├───────────────┤
│ ⇄ 回聊天       │  ← 底部：模式切换（与聊天侧栏底部行对称）
└───────────────┘
```

- 选中话题 → 主界面切到该话题的任务工作台；
- 空态（未选话题/新话题）：主界面为干净草稿态（工作区 chip + 输入区，
  发首条消息开始任务）；
- 状态色板（全局统一）：running=primary（呼吸）、waitingApproval=橙、
  waitingInput=蓝、paused=灰、done=绿、failed=红、cancelled=灰删除线。

### 智能体编辑页（新建/编辑档案）

已拍板：**复用普通聊天的助手设置页**（名称/图标/系统提示/模型与参数等
共用项直接用现有页面组件），顶部加一个切换入口进「智能体专用」独立页，
只放智能体独有项：

- **工具集勾选**：文件编辑 / 终端 / 网络搜索 / 知识库 / 技能（分组开关，
  决定该智能体每轮可见的工具清单）；
- 绑定工作区（一个工作区对应一个智能体，话题直接继承；未绑定时
  草稿态提醒去这里选）；
- 权限模式偏好（P1）。

内置预设档案（编程/网络研究/文档）不可删可复制。

### 新建话题（干净空态，已拍板，替代早期"底部弹层填表单"方案）

新建话题不弹表单——像普通聊天一样直接进入干净空白的草稿态：

```
┌────────────────────────────────┐
│          💻                     │
│   [📁 aetherlink-app]           │ ← 工作区 chip：继承自当前智能体
│   发送第一条消息开始任务        │   （未绑定时提醒去智能体设置选）
│ ┌──────────────────────────┐  │
│ │ 输入区（与工作台同款）     │  │ → 发送首条消息 = 建话题+引擎启动
│ └──────────────────────────┘  │
└────────────────────────────────┘
```

模式（Code/Ask/Plan）在输入区上沿快切（默认 Code）；模型跟随聊天当前
选择，输入区模型 chip 可换。工作区不在这里选——绑在智能体上（上节），
未绑定工作区的智能体不能开始任务（草稿态提醒去设置）。

## 四、任务工作台（主界面本体，核心页面）

两页全屏横滑（工作区页同款 `PageView` + 页指示器，已拍板：2 页而非 3 页），
默认落在左页：

- **左页 = 事件流**（主视图）；计划不单独占页——顶部计划纪要条点开看
  完整 todo，任务信息卡（工作区/模型/模式/token 累计）也收进这个面板；
- **右页 = 工作台**：顶部小 tab 切换「终端 / AI 正在看的文件 / 改动 diff」，
  右滑一下就能看它正在干嘛。

### 4.1 左页：事件流（主视图）

```
┌────────────────────────────────┐
│ ☰侧栏   修复登录页崩溃     ⋯    │ ← 顶栏：侧栏按钮 + 话题标题 + 菜单(改名/删除)
│ ● 运行中 · 第12轮 · 8.4k · 6m  │ ← 状态条（常驻，状态色）
├────────────────────────────────┤
│ ▤ 计划 3/5 ▸                   │ ← 计划纪要条（可折叠；点开=完整todo
├────────────────────────────────┤    +任务信息卡，见 4.2）
│ │                              │
│ ●  我先看一下崩溃日志…          │ ← 助手叙述：无气泡、贴左的正文段落
│ │                              │
│ ○  read_file lib/login.dart    │ ← 工具行（collapsed 单行）：
│ │   ↳ 234 行 · 0.4s ✓          │    图标+工具名+关键参数+结果摘要
│ ○  terminal_execute            │    点击 → 底部抽屉看完整参数/输出
│ │   $ flutter test · 失败 ✗    │    （复用现有命令/diff 展示件）
│ │                              │
│ ┌─ ⚠ 等待授权 ────────────┐   │ ← 审批卡（内嵌事件流）：
│ │ write_to_file login.dart  │   │    摘要+diff预览入口
│ │ [批准] [拒绝] [白名单▾]   │   │    白名单▾=本任务不再问/永久白名单
│ └──────────────────────────┘   │
│ ── ✂ 已压缩 41 条早期事件 ──    │ ← compaction 分隔线（点开看摘要）
│ ▸ 工作了 36s · 14 个操作 · +17−7│ ← 已完成工作段自动折叠成摘要块（见下）
│ │                              │
├────────────────────────────────┤
│ ⌨ Code ▾         GLM-4.6 ▾  │ ← 输入条上沿 chips：模式切换 + 模型选择
│ ＋ [ 追加指令…     ] ➤/⏸      │ ← 底部控制条（＋附件；发送钮按输入状态
└────────────────────────────────┘    变形，见 §五）
```

### 输入区（已拍板）

- **上沿 chips 行**（对标 Cursor/Devin 底部条）：
  - 左：**模式快切**「⌨ Code ▾」，点开三选，与权限模式一一对应：
    · **Code** —— 执行模式（写/终端全能力，走审批+白名单）；
    · **Ask** —— 只问答（仅只读工具，不改任何东西）；
    · **Plan** —— 只读规划（先出完整方案，确认后一键转 Code 继续执行）。
    话题中途可切；Plan → Code 是最常用流（"先给方案 → 行，干吧"）；
  - 右：**模型选择**（复用聊天模型选择器，不放顶栏——顶栏已有侧栏/
    标题/状态，模式与模型都是"发指令前顺手确认"的东西，贴输入框最顺）。
- **＋按钮**弹面板：图片/文件上传、引用工作区文件（选文件插入路径
  引用）、粘贴长文本转附件；复用聊天已有附件能力，附件随指令入队进
  `UserMessageEvent`。

### 工作段折叠（已拍板：Devin 式分组 + Claude Code 式实况）

- **执行中**：当前工作段实时展开，工具行逐条滚出（实况，不黑盒）；
- **一段工作完成后**（以下一次向用户叙述/提问/审批为分界）：这一串
  工具调用自动折叠成一个摘要块：「▸ 工作了 36s · 14 个操作 · +17−7」
  （耗时 / 操作数 / 文件增删行 / 有失败则 ✗ 标红），点开可回看每一行；
- 时间线因此只剩「叙述 → 工作块 → 叙述 → 审批」的节奏，长任务上百个
  工具调用也不会冲爆小屏时间线。

要点：

- **时间线左轨**：一条纵线贯穿，节点形状/颜色区分事件类型
  （●助手文字 ○工具 ⚠审批 ✂压缩 ◆状态变化），扫一眼就能读出执行节奏；
- **工具行默认收起**：一行 = 名称 + 关键参数（路径尾段/命令）+ 结果摘要
  （行数/耗时/✓✗）。展开走底部抽屉而非原地撑开，保持时间线密度；
- **大输出**：抽屉里只显截断内容 + 「查看全文」（全文落盘文件）；
- 流式中的助手文字/运行中的工具行有进行态动效（光标闪烁/转圈）；
- waitingInput 时模型的提问渲染为高亮问题卡，回答走底部输入条。

### 4.2 计划纪要面板（事件流顶部，取代原独立计划页）

- **计划**：完整 todo 清单（`PlanUpdateEvent` 驱动），完成打勾、当前项高亮；
- **任务信息卡**：工作区（点击可跳工作区页）、模型、模式、创建时间、
  token/轮数累计。

### 4.3 右页：工作台（顶部小 tab）

- P0：**终端**——任务绑定工作区的 AI 终端会话（复用现有多会话终端组件），
  实时看命令在跑什么；
- P0：**AI 正在看的文件**——跟随 read_file/写入实时定位打开（复用编辑器
  只读态）：工具行点击是回看，这里是实况，两入口互补；
- P1：**改动 diff** tab + 改动文件清单；
- 无可跑命令后端（纯 SAF 工作区）时终端 tab 显示说明空态。

## 五、审批与打断体验

- 审批发生 → 状态条变橙、事件流滚到审批卡、轻震动（复用 haptics）；
  App 在任务页外 → 任务列表卡片高亮；系统通知 P1；
- 审批卡三个动作：**批准**（本次）/ **拒绝**（写入拒绝结果，模型继续）/
  **白名单▾**（"本任务内不再询问此工具" = 运行级宽限；
  "永久免授权" = 写入现有 `tool_auth_policy` 白名单）；
- 超时不自动拒绝：任务转 paused（详见架构稿 §4.2），卡片保留可补答。

### 打断交互（已拍板：输入框状态驱动，软硬两档）

底部控制条按输入框是否有文字变形：

- **有文字**：发送钮点击不直接发，弹小面板三选：
  1. **排队**（默认/温和）：不打断当前工具，入队为 `UserMessageEvent`，
     下一轮 LLM 调用生效，事件流立即回显"已排队"态；
  2. **立即打断并发送**（强制）：中止当前工具/流式输出，指令作为最高
     优先级注入，模型下一轮必须先响应它——适合"方向偏了快停"；
  3. **继续编辑**：收起面板回输入框。
- **无文字**：同一按钮显示为中断 ⏸：点一下 = 暂停（当前工具跑完即停，
  可继续）；长按或再点弹「强制终止」二次确认（不可恢复）。

误触不会毁掉任务：硬打断前都有面板/确认一层缓冲。

## 七、P0 UI 裁剪

P0 必做：侧边栏（智能体/话题两级，含新建话题弹层）、内置 1 个「编程」
预设档案、中页事件流（含审批卡/控制条）、左页计划清单、模式切换 + 持久化
（含 last-active 智能体/话题恢复）。
P0 缓做：智能体自建/编辑页（先只用预设档案）、右页工作台（先放"去终端页
看"跳转占位）、改动文件清单、系统通知、滑动删除动效。
