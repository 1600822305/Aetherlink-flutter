## 一、现状总览（已经做得不错的）

- 三页横滑结构（文件树 / 编辑器 / 终端），会话自动恢复（工作区 + 打开的 tab + 活动 tab）
- 后端抽象干净：SAF / PRoot / SSH 统一接口，capabilities 门控 UI
- 文件树：懒加载 + 缓存、watch 事件去抖增量刷新、IDE 式"跟随活动文件"reveal、保护挂载点
- 编辑器：行号栏、当前行高亮、状态栏（行:列/选区）、查找替换（正则/大小写）、捏合缩放、Tab 缩进 + 自动缩进、外部修改冲突 banner、大文件/二进制/图片分级打开策略、虚拟化只读视图
- 文件操作：新建/重命名/移动/复制/删除，全部走目录选择器与确认对话框
- 终端：多 tab PTY、AI 会话接管视图

整体骨架是专业的，缺的主要是"小功能密度"和细节打磨。

## 二、从普通用户角度：缺失的小功能

### 文件树
1. ✅ ~~无文件名过滤/搜索框~~——已接入 `searchFiles`（文件名/内容/正则，5941937）。
2. ⬜ **看不到隐藏文件开关**——`WorkspaceEntry.isHidden` 已有字段，树上没有"显示隐藏文件"切换。
3. ⬜ **无排序选项**——不能按名称/修改时间/大小/目录优先排序（当前完全依赖后端返回顺序）。
4. ⬜ **长按菜单缺"打开方式/详情"**——没有"属性/详情"（大小、修改时间、完整路径）、没有"复制路径"、没有"用其他应用打开/分享"（手机场景很常用）。
5. ⬜ **不能多选**——批量删除/移动/复制要一个一个来。
6. ⬜ **新建文件无模板/无自动打开**——创建后不自动在编辑器打开，用户要再点一次。
7. ⬜ **删除无撤销**——没有回收站/undo snackbar，SAF 上误删不可逆。
8. ⬜ **重名冲突体验**——move/copy 冲突只会抛错 toast，没有"覆盖/保留两者/重命名"选择。
9. ⬜ **文件图标很少**——只识别 dart/md/yaml/json/图片，js/ts/py/html/css/压缩包等都是通用图标。

### 编辑器
10. ⬜ **无撤销/重做按钮**——TextField 自带 undo 栈但没暴露 UI（软键盘用户基本用不到 Ctrl+Z）。
11. ⬜ **保存无快捷路径**——dirty 状态下返回/切 tab 有守卫（好），但没有"全部保存"。
12. ⬜ **无"跳转到行"**——底层 `open(entry, line:)` 已就绪（0e65dd1），缺状态栏点击弹"跳到行"输入框的 UI。
13. ⬜ **查找历史不保留**——每次打开 find bar 都是空的。
14. ⬜ **无最近文件列表**——tab 关掉就找不回，缺"最近打开"。
15. ⬜ **图片预览缺缩放信息/分享**；二进制/超大文件占位页没有"用外部应用打开"逃生口。
16. ⬜ **自动保存选项缺失**——手机上切走 app 容易丢编辑（虽然有 dirty 守卫）。

### 终端
17. ✅ ~~无额外按键行~~——已加 Esc/Tab/Ctrl/Alt/方向键（长按连发）/Home/End/PgUp/PgDn 工具条，粘滞 Ctrl/Alt（30b78a9）。
18. ⬜ **无字体大小调节/复制粘贴按钮**（编辑器有捏合缩放，终端页没有对应能力）。
19. ⬜ **无命令历史/常用命令收藏**。

## 三、从专业 IDE 角度：缺失的能力

### P0（对"IDE"定位影响最大）
1. ✅（部分）~~语法高亮完全没有~~——只读视图已按行高亮（7ffd4f4）；**编辑态 TextField 仍是单色**，需要自绘高亮编辑器（大工程，未做）。
2. ✅ ~~全局搜索（跨文件内容搜索）~~——搜索面板 + 匹配行预览 + 跳行（d89a65e），SSH 远端 grep 加速（eeaf4b8）。
3. ✅ ~~打开文件后跳转到指定行~~——`open(entry, line:)` 一等 API（0e65dd1）。

### P1
4. ✅ ~~括号匹配/自动闭合~~——自动闭合配对/包裹选区/跳过重复/退格删对 + 括号匹配高亮（48ae9f6）。
5. ✅（部分）~~多行 Tab 缩进~~——多行 Tab/Shift+Tab 整体缩进已修（48ae9f6）；**多光标/列选择未做**（需重写 TextField 编辑框架，成本极高）。
6. ✅ ~~注释切换~~——Ctrl+/ + 工具条按钮，按语言选前缀（86180a8）。
7. ✅（部分）~~文件对比/保存冲突~~——冲突双向 diff 视图 + 二选一（86180a8）；**逐 hunk 三方合并未做**（需 base/local/disk 三方与按块挑选 UI）。
8. ⬜ **Git 集成缺失**——树上无修改状态染色（M/A/U），无 diff、无 blame。SSH/PRoot 后端有 exec，最小可行版（status 染色 + diff 查看）成本不高。
9. ⬜ **编码/换行符**——一律按 UTF-8 读写，状态栏不显示也不能切换编码（GBK 文件会乱码且保存会破坏），CRLF/LF 不显示不可切换。

### P2
10. ⬜ **符号导航/大纲**——无 outline、无 go-to-definition（可先做基于正则的当前文件符号列表）。
11. ⬜ **代码折叠**。
12. ⬜ **面包屑路径**——header 只显示截断的 readable path，不能点某一段跳转目录。
13. ⬜ **拖拽 tab 排序 / 拖拽文件到目录移动**。
14. ⬜ **编辑器设置持久化**——字体大小是每个 tab 的局部 state，换 tab/重进就丢；无 tab 宽度（2/4 空格）、无软换行开关（长行才被动降级 wrap）。
15. ⬜ **终端与编辑器联动**——终端里 `path:line` 输出不能点击打开文件；无"在终端中打开当前文件所在目录"（跳行 API 已就绪，联动只差终端侧解析）。
16. ⬜ **任务/运行配置**——无常用命令快捷运行（可与 19 条合并做"命令收藏"）。

## 四、剩余建议落地顺序（性价比排序）

1. 文件树小功能包：隐藏文件开关 + 复制路径/详情菜单 + 新建后自动打开（纯 UI，低成本高体感）
2. 编辑器：撤销/重做按钮 + 状态栏点击跳行（跳行地基已就绪）
3. Git status 染色 + diff 查看（限 canExec 后端，diff 视图组件已有可复用）
4. 终端字体调节/复制粘贴按钮；终端 `path:line` 点击打开（跳行 API 已就绪）
5. 删除软保护（不可恢复提示/undo）、重名冲突"覆盖/保留两者"
6. 编辑器设置持久化（字体大小/tab 宽度/软换行）
7. 编码/换行符显示与切换
8. 大工程（另行立项）：编辑态语法高亮、多光标/列选择、逐 hunk 三方合并、符号大纲/代码折叠
