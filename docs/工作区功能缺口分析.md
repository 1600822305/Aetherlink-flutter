## 一、现状总览（已经做得不错的）

- 三页横滑结构（文件树 / 编辑器 / 终端），会话自动恢复（工作区 + 打开的 tab + 活动 tab）
- 后端抽象干净：SAF / PRoot / SSH 统一接口，capabilities 门控 UI
- 文件树：懒加载 + 缓存、watch 事件去抖增量刷新、IDE 式"跟随活动文件"reveal、保护挂载点
- 编辑器：行号栏、当前行高亮、状态栏（行:列/选区）、查找替换（正则/大小写）、捏合缩放、Tab 缩进 + 自动缩进、外部修改冲突 banner、大文件/二进制/图片分级打开策略、虚拟化只读视图
- 文件操作：新建/重命名/移动/复制/删除，全部走目录选择器与确认对话框
- 终端：多 tab PTY、AI 会话接管视图

整体骨架是专业的，缺的主要是"小功能密度"和细节打磨。

## 三点五、Git 集成完整版规划（未做，待立项）

最小版（status 染色 + 只读 diff）已落地，见 §三.8。完整版目标是 VSCode 源代码管理面板级别的能力，按性价比分阶段：

### 阶段 1：暂存 + 提交 + 推送闭环（最常用）
- ⬜ 变更列表面板：按暂存/未暂存分组展示 status 文件，点击看 diff（复用只读 diff 面板）
- ⬜ 暂存/取消暂存单个文件（`git add <file>` / `git restore --staged <file>`）与全部
- ⬜ 丢弃改动（`git restore <file>` / 未跟踪文件删除，需二次确认，不可恢复）
- ⬜ 提交：输入提交信息 + `git commit`；amend 上一条
- ⬜ 推送/拉取：`git push` / `git pull`，失败时展示 stderr（认证问题提示走 SSH 配置）

### 阶段 2：分支与历史
- ⬜ 分支列表/切换/新建/删除（`git branch` / `git switch`），头部显示当前分支 + ahead/behind
- ⬜ 提交历史查看（`git log` 列表 + 单提交 diff）；单文件历史
- ⬜ stash：保存/恢复/列表

### 阶段 3：精细化
- ⬜ 逐 hunk 操作：diff 面板按块「暂存这块/丢弃这块」（`git apply --cached` 或 `add -p` 脚本化）
- ⬜ blame：行级归属展示（编辑器 gutter 或单独面板，`git blame --porcelain`）
- ⬜ 编辑器行号栏改动标记（增/改/删色条，基于 `git diff -U0` 行区间）
- ⬜ 冲突解决 UI：merge/rebase 冲突文件逐块选择 ours/theirs（可与「三方合并」大工程合并立项）

### 实现约束（沿用最小版）
- 全部走 `WorkspaceBackend.exec`，限 canExec 后端（SSH/PRoot/Termux），SAF 不适用
- 命令输出解析放 application 层纯 Dart（沿 `workspace_git_status.dart` 模式），UI 在 presentation/mobile
- 路径参数一律 `shellQuoteArg` 引用；写操作（restore/丢弃/push）需确认对话框
- 每步操作后刷新 `gitStatusProvider`
