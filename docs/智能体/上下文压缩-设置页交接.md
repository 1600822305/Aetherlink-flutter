## 1. 现状盘点（用户现在怎么用压缩）

### 1.1 现有用户入口（仅两处，都在侧栏「智能体设置 tab」）

文件：`lib/features/agent/presentation/mobile/sidebar/tabs/agent_settings_tab.dart`
「上下文」分组里：

1. **会话上下文长度**（`_SelectRow`，32k/64k/128k/200k/256k/1M）——写
   `AgentUiSettings.contextLimit`（`agent_providers.dart`，持久化），是 token 触发
   路径的窗口来源。
2. **立即压缩**（`_EntryRow` → `_compactNow` 确认弹窗 + toast）——手动压缩
   （升级 ⑤，`AgentTaskRunner.compactNow`）。
3. 只读展示：分组副标题「上限 xx · 当前已用 xx」+「当前话题已用 / 剩余」行。

### 1.2 目标承载页面（用户已拍板）

智能体首页三点菜单（`agent_home_page.dart`）→「设置」→
`${AppRouter.settingsPath}?mode=agent` → `SettingsPage`（`settings_page.dart`）
agent 模式的 body 是 **`AgentAppSettingsBody`**
（`lib/features/agent/presentation/mobile/agent_app_settings_body.dart`），
**目前还是「智能体设置项待设计」的占位页**。

👉 用户要求：在这个页面里新增「上下文压缩」入口，点进**独立的压缩设置页**。
（不是加在三点菜单本身。）

### 1.3 引擎侧现状：可调什么、现在硬编码在哪

| 参数 | 现值 | 位置 | 现在可配吗 |
|---|---|---|---|
| 上下文窗口（token 触发用） | 默认 128k | `AgentUiSettings.contextLimit` → `AgentBudget.contextLimitTokens` | ✅（唯一可配项） |
| 摘要输出预留 | 20k（min(20k, 窗口/4)） | `agent_compaction_trigger.dart` | ❌ 常量 |
| 自动压缩触发比例 | 有效窗口 × 92% | `agent_compaction_trigger.dart` | ❌ 常量 |
| 字符回退触发阈值 | 120000 字符 | `AgentBudget.compactionTriggerChars` | ❌ 构造默认值 |
| 压缩保留量 | 40000 字符 | `AgentBudget.compactionKeepChars` | ❌ 构造默认值 |
| microcompact 触发阈值 | 80000 字符（`kMicroCompactTriggerChars`） | `agent_microcompact.dart` / `AgentBudget.microCompactTriggerChars` | ❌ 常量 |
| microcompact 最小清除长度 / 近期保护条数 | 2000 字符 / 5 条 | `agent_microcompact.dart` | ❌ 常量 |
| 预警比例 | 触发阈值 × 90% | `agent_compaction_guard.dart` | ❌ 常量 |
| 熔断连续失败次数 | 3 | `agent_compaction_guard.dart` | ❌ 常量 |
| 自动压缩总开关 | 无（永远开） | — | ❌ 不存在 |
| microcompact 开关 | 无（永远开） | — | ❌ 不存在 |
| 压缩用模型 | 跟随任务当前模型（`_resolveModel`） | `agent_runtime_access.dart` `summarizeForCompaction` | ❌ 不可单独指定 |

⚠️ **一致性硬点**：重放侧 `_replayMessages`（`agent_runtime_access.dart`）当前直接用
常量 `kMicroCompactTriggerChars`，引擎侧用 `budget.microCompactTriggerChars`。一旦
microcompact 阈值可配，**两侧必须读同一来源**（否则引擎核算的字符量与真实进模型的
视图不一致）。这是本任务最容易引 bug 的点。

### 1.4 设置持久化的现成模式（照抄即可）

- 参考实现：`agent_hooks_settings.dart` 的 `AgentDisableAllHooksNotifier`
  （异步 hydrate + `_pendingWrite` 防覆盖 + encode/decode 纯函数可单测）。
- 独立页面参考：`agent_hooks_page.dart` 的 `showAgentHooksPage`
  （`PageRouteBuilder` 零动画推入）。

## 2. 目标设计

### 2.1 新模块拆分（别堆一个文件）

1. **`lib/features/agent/application/agent_compaction_settings.dart`**（设置模型 + 持久化）
   - `AgentCompactionSettings`：不可变数据类 + copyWith + encode/decode 纯函数
     （JSON 单键存储，key 如 `agent_compaction_settings`，坏数据回退默认值）。
   - Notifier（照 `AgentDisableAllHooksNotifier` 的 hydrate/pendingWrite 模式）。
   - 字段（v1 建议，全部有默认值 = 当前行为不变）：
     - `autoCompactEnabled`（bool，默认 true）——关掉后自动压缩不触发，
       预警照发（对标 CC 的 autocompact 开关：关掉时仍按有效窗口告警）；
       手动「立即压缩」不受影响。
     - `microCompactEnabled`（bool，默认 true）。
     - `triggerRatio`（double，默认 0.92，UI 用档位 85%/90%/92%/95%，不做自由输入）。
     - `keepChars`（int，默认 40000，档位 20k/40k/60k）。
     - `microCompactTriggerChars`（int，默认 80000，档位 60k/80k/100k）。
   - **v1 不做**：预警比例、熔断次数、min 清除长度、近期保护条数（保持常量，
     设置页只读展示即可）；压缩专用模型（涉及模型选择器接线，单独评估）。
2. **`lib/features/agent/presentation/mobile/agent_compaction_settings_page.dart`**（独立设置页）
   - `showAgentCompactionSettingsPage(BuildContext)`（零动画，照 hooks 页）。
   - 分组建议：
     - 「自动压缩」：总开关、触发比例档位、保留量档位；只读展示当前
       窗口/阈值换算结果（如 128k 窗口 → 约 99k tokens 触发）。
     - 「轻量清理（microcompact）」：开关、阈值档位；说明文案（清旧工具输出、
       不调 LLM、事件流原文保留）。
     - 「保护与熔断」（只读展示）：预警 90%、连续失败 3 次熔断、近期 5 条保护。
     - 「手动压缩」：「立即压缩」入口（复用 `AgentTaskRunner.compactNow` 与
       settings tab 同款确认弹窗逻辑——可把 `_compactNow` 提炼成共享 helper，
       两处调用；没有活动任务时该行禁用并说明）。
   - 页内文案强调语义：压缩只影响进模型上下文，事件流原文保留可审计。
3. **`AgentAppSettingsBody` 改造**：占位页替换为分组入口列表（照聊天设置页
   `SettingGroup`/`SettingItem` 的形态或自建等价轻量组件），v1 至少一项
   「上下文压缩」→ `showAgentCompactionSettingsPage`。（Hooks/技能/MCP/权限
   入口是否也收进来由实施者判断，非本任务必须。）

### 2.2 接线点（引擎读取设置）

- `AgentTaskRunner` 构建 `AgentBudget` 处（`agent_task_runner.dart`，主任务与
  subagent 两处）：从 `agentCompactionSettingsProvider` 读值填
  `compactionTriggerChars`（由 ratio 推导或保留字符阈值语义）、`compactionKeepChars`、
  `microCompactTriggerChars`；`AgentBudget` 若需加字段（如 `autoCompactEnabled`、
  `compactionTriggerRatio`）就加，别用全局可变状态。
- `agent_compaction_trigger.dart`：`compactionTriggerTokens` 接受 ratio 参数
  （默认 0.92 保持向后一致）。
- 引擎 `_maybeCompact`：`autoCompactEnabled=false` 时跳过自动触发（forced 手动
  信号仍执行）；`microCompactEnabled=false` 时两侧都不做 micro。
- **重放侧一致性**（§1.3 的硬点）：`_replayMessages` 需要拿到与引擎相同的
  micro 设置。建议：`AgentLlmContext`（或 runTurn 入参）带上生效的
  microTriggerChars/开关，或 di 侧直接读同一 provider——实施者选一种，
  必须保证同一任务同一轮两侧数值一致，并写单测/注释锁住。

### 2.3 状态可见性（可选加分项）

- settings tab「上下文」分组保留现有三项不动（快捷入口），可加一行
  「压缩设置」跳转新页，避免功能藏太深。

### 实现记录（2026-07-20，A–E 一次性完成）

- **A 设置模型**：新增 `lib/features/agent/application/agent_compaction_settings.dart`——
  `AgentCompactionSettings`（autoCompactEnabled / microCompactEnabled /
  triggerRatio / keepChars / microCompactTriggerChars，全部默认值 = 现行为）+
  encode/decode 纯函数（单键 JSON，key `agent_compaction_settings`，坏数据
  逐字段回退默认）+ Notifier（hydrate/pendingWrite 同 hooks 设置）。字符回退
  触发阈值 `compactionTriggerChars` 由 ratio 等比推导（默认比例下 = 120000，
  与既有 AgentBudget 默认一致）。单测 `agent_compaction_settings_test.dart`。
- **B 引擎接线**：`AgentBudget` 新增 `autoCompactEnabled` / `microCompactEnabled` /
  `compactionTriggerRatio`；`compactionTriggerTokens` / `shouldTriggerCompaction` /
  `isNearCompactionThreshold` 接受 `triggerRatio` 参数（默认 0.92 向后一致）；
  引擎 `_maybeCompact`：关自动压缩后阈值不自动触发（forced 手动信号仍执行），
  已超阈值时预警改提示「自动压缩已关闭，可手动压缩」；micro 关闭时两侧都
  不做。**重放侧一致性（§1.3 硬点）**：`AgentLlmContext` 新增
  `microCompactEnabled` / `microCompactTriggerChars`，引擎每轮从 budget 填入，
  `_replayMessages` 改读 context 值（不再直用常量），单测锁住。task runner
  `_budgetFromSettings` 统一从 provider 组装主任务与 subagent 预算；
  `compactNow` / `runManualCompaction` 同样读设置。引擎单测新增 2 例
  （关自动压缩不触发 + 预警一次；micro 设置随 context 传重放侧）。
- **C 设置页 UI**：新增 `agent_compaction_settings_page.dart`：
  `showAgentCompactionSettingsPage`（零动画，同 hooks 页）；分组：自动压缩
  （开关 + 比例 85/90/92/95% + 保留量 20k/40k/60k + 窗口换算只读）、轻量清理
  （开关 + 阈值 60k/80k/100k）、保护与熔断（只读：预警 90%、熔断 3 次、
  近期 5 条、摘要预留 20k）、手动压缩（无可压缩话题时禁用）。手动压缩确认
  弹窗提炼为共享 `confirmAndCompactNow`，settings tab 改调同一 helper。
- **D 设置入口**：`AgentAppSettingsBody` 占位页换成入口列表（v1 一项
  「上下文压缩」）；另在侧栏 settings tab「上下文」组加「压缩设置」快捷跳转
  （§2.3 加分项）。无 chat import。
