# 新增内置宠物（Buddy 物种）指南

本文档是一份可照抄执行的操作手册：按顺序做完每一步，就能给电子宠物
（buddy 功能）新增一个内置物种。已按本流程加过的物种：奶龙
（`nailong`）、蓝咕咕（`blueGuga`）、腼腆奶龙（`shyNailong`）。

## 0. 背景知识（务必先读）

### 0.1 相关文件一览

| 文件 | 作用 |
| --- | --- |
| `lib/features/buddy/domain/buddy_types.dart` | `BuddySpecies` 物种枚举（含中文名） |
| `lib/features/buddy/domain/pixel_arts/pixel_art.dart` | `BuddyPixelArt` 数据类型与共享常量 |
| `lib/features/buddy/domain/pixel_arts/<物种>.dart` | 每个物种一个像素图文件 |
| `lib/features/buddy/domain/pixel_arts/pixel_arts.dart` | 注册表 `kBuddyPixelArts`（物种 → 像素图） |
| `lib/features/buddy/domain/buddy_engine.dart` | 种子 → 宠物骨架的确定性生成（物种池按种子版本选取） |
| `lib/features/buddy/application/buddy_controller.dart` | 孵化入口（决定新种子的版本前缀） |
| `test/features/buddy/buddy_engine_test.dart` | 物种池 / 像素图格式的单测 |

### 0.2 像素图格式（`BuddyPixelArt`）

- `rows`：**128 个字符串，每个长度 128**（128×128 网格）。
- `palette`：字符 → ARGB 颜色（`0xFFRRGGBB`）。
- 字符约定：
  - `.` = 透明（不需要出现在 palette 里）；
  - `E` = 眼睛（渲染器负责眨眼动画和眼睛高光，**每个物种必须至少
    有一个 `E` 像素**，palette 里给 `E` 配眼睛颜色）；
  - `p` = 腮红（可选，共享粉色 `kBuddyBlushColor`）；
  - 其余字符随意（惯例用 `a-z`、`A-Z`，避开 `E` 和 `p`）。
- 单测会强制校验：行数 128、每行长 128、所有非 `.` 字符都在
  palette 里、必须有 `E`。

### 0.3 种子版本规则（最容易踩坑，必读）

宠物由种子字符串确定性生成（`rollBuddy`），物种是从「物种池」里按
随机数取模抽的。**池子大小一变，同一种子抽到的物种就变**，所以：

1. `BuddySpecies` 枚举**只能追加在末尾**，绝不能插到中间或重排。
2. 每个「已发布」的种子版本，其物种池大小被永久冻结在
   `buddy_engine.dart` 里：
   - 旧种子（无版本前缀）→ 前 18 种（`_kSpeciesCountV1`）；
   - `v2-` 前缀 → 前 19 种（`_kSpeciesCountV2`）；
   - `v3-<物种名>-` 前缀 → 自选物种（池子大小无所谓）；
   - `v4-` 前缀（当前随机孵化在发的版本）→ 全部物种。
3. 新增物种时怎么处理，二选一：
   - **当前最新种子版本还没随正式版发布**（一般是这种情况）：直接把
     新物种追加到枚举末尾即可，v4 池子自动包含它，什么都不用改。
   - **当前最新种子版本已经发布给用户**：必须冻结现有池子并升版本。
     例如从 v4 升 v5：在 `buddy_engine.dart` 加
     `const int _kSpeciesCountV4 = <当前物种数>;`，`rollBuddy` 里给
     `v4-` 种子用 `sublist(0, _kSpeciesCountV4)`，`_isLegacySeed` 加
     `!seed.startsWith('v5-')`，`buddy_controller.dart` 的 `hatch()`
     里随机种子改发 `'v5-...'`，并补一条「v4 抽不到新物种」的单测。
     参照 git 历史提交 `03132f8f`（蓝咕咕，v2→v4 升版本的完整示例）。

## 1. 准备 128×128 像素图

两条路线，选其一。

### 路线 A：从 Codex Pets 官方精灵图量化（推荐，效果最好）

codex-pet.org 上的宠物都有公开精灵图 API：

```text
元数据:   https://codex-pet.org/api/pets/<宠物id>/files/pet.json
精灵图:   https://codex-pet.org/api/pets/<宠物id>/files/spritesheet.webp
```

`<宠物id>` 从宠物页 URL 取，例如
`https://codex-pet.org/zh/pets/blue-guga/` 的 id 是 `blue-guga`。

精灵图是固定 **8 列 × 9 行** 的网格（每行一种动画状态、每行 8 帧），
**待机动画是第 0 行，取第 0 行第 0 帧**做静态像素化。

用下面的 Python 脚本（需要 Pillow：`pip install pillow`）生成 dart
文件。脚本做的事：裁掉透明边 → 等比缩到高 128 → 水平居中放进
128×128 画布 → 标记眼睛像素为 `E` → 其余像素中值切分量化成 ≤18 色 →
输出 `BuddyPixelArt` 源码和一张预览图。

```python
# gen_pet.py — 用法: python3 gen_pet.py
# 修改下面 4 个配置后运行；生成 <var>.dart 和 <var>_preview.png。
from PIL import Image
import urllib.request

PET_ID = 'blue-guga'          # ① codex-pet.org 宠物 id
VAR = 'blueGugaArt'           # ② dart 变量名（<物种名>Art）
HEADER = [                    # ③ 文件头注释（写清来源和外观特征）
    '// 蓝咕咕：以 Codex Pets 官方 Blue Guga 精灵图（待机第 1 帧）为参考',
    '// 量化的 128×128 高清像素版——冰蓝企鹅连体衣、棕发圆脸、大眼睛。',
]
EYE_COLOR = 0xFF2E2C36        # ④ E 字符的眼睛颜色（从原图取深色瞳色）

# ⑤ 眼睛判定：先跑一遍脚本看预览图，再回来按坐标圈出双眼区域。
#    常用判定：绿眼睛用 g>r+15；深色瞳孔用亮度 lum<110~150。
def is_eye(x, y, p):
    r, g, b, a = p
    lum = 0.3 * r + 0.6 * g + 0.1 * b
    if not (44 <= y <= 58):                      # 眼睛所在的行范围
        return False
    if not (44 <= x <= 62 or 68 <= x <= 86):     # 左右眼的列范围
        return False
    return lum < 110

SIZE, NCOLORS = 128, 18
CHARS = 'abcdfghijklmnoqrstuvwxyzABCDFGHIJKLMNOQRSTUV'  # 避开 E/p

url = f'https://codex-pet.org/api/pets/{PET_ID}/files/spritesheet.webp'
urllib.request.urlretrieve(url, 'sheet.webp')
sheet = Image.open('sheet.webp').convert('RGBA')
fw, fh = sheet.width // 8, sheet.height // 9
frame = sheet.crop((0, 0, fw, fh))               # 待机第 1 帧
frame = frame.crop(frame.getbbox())              # 裁透明边
h = SIZE
w = round(frame.width * h / frame.height)
small = frame.resize((w, h), Image.LANCZOS)
canvas = Image.new('RGBA', (SIZE, SIZE), (0, 0, 0, 0))
canvas.paste(small, ((SIZE - w) // 2, 0))
px = canvas.load()

eye = [[px[x, y][3] >= 128 and is_eye(x, y, px[x, y])
        for x in range(SIZE)] for y in range(SIZE)]
opaque = [(x, y, px[x, y][:3]) for y in range(SIZE) for x in range(SIZE)
          if px[x, y][3] >= 128 and not eye[y][x]]
strip = Image.new('RGB', (len(opaque), 1))
strip.putdata([p[2] for p in opaque])
q = strip.quantize(colors=NCOLORS, method=Image.MEDIANCUT,
                   dither=Image.Dither.NONE)
pal = q.getpalette()
colors = [tuple(pal[i * 3:i * 3 + 3]) for i in range(NCOLORS)]
grid = [[None] * SIZE for _ in range(SIZE)]
for (x, y, _), i in zip(opaque, q.getdata()):
    grid[y][x] = i
used = sorted({i for row in grid for i in row if i is not None})
cmap = {i: CHARS[n] for n, i in enumerate(used)}

rows = []
for y in range(SIZE):
    line = ''
    for x in range(SIZE):
        if eye[y][x]:
            line += 'E'
        elif grid[y][x] is None:
            line += '.'
        else:
            line += cmap[grid[y][x]]
    rows.append(line)
palette = {'E': EYE_COLOR}
for i, ch in cmap.items():
    r, g, b = colors[i]
    palette[ch] = 0xFF000000 | (r << 16) | (g << 8) | b

lines = HEADER + ['', "import 'pixel_art.dart';", '',
                  f'const BuddyPixelArt {VAR} = BuddyPixelArt(', '  rows: [']
lines += [f"    '{r}'," for r in rows]
lines += ['  ],', '  palette: {']
lines += [f"    '{ch}': 0x{palette[ch]:08X}," for ch in sorted(palette)]
lines += ['  },', ');']
open(f'{VAR}.dart', 'w').write('\n'.join(lines) + '\n')

out = Image.new('RGBA', (SIZE, SIZE), (0, 0, 0, 0))
op = out.load()
for y in range(SIZE):
    for x in range(SIZE):
        ch = rows[y][x]
        if ch != '.':
            v = palette[ch]
            op[x, y] = ((v >> 16) & 255, (v >> 8) & 255, v & 255, 255)
out.resize((512, 512), Image.NEAREST).save(f'{VAR}_preview.png')
print('palette size:', len(palette))
```

操作要点：

1. 第一遍先随便填 `is_eye`（比如恒 `False`）跑通，打开预览图确认整体
   效果、找到双眼的像素坐标范围。
2. 按坐标改好 `is_eye` 再跑，直到预览里眼睛区域完整、没把头发/轮廓
   误标成 `E`。眼睛判定宁可只标瞳孔核心（眨眼动画只遮 `E` 像素）。
3. palette 总数（含 E）≤ 20 为宜；小细节（嘴、纽扣）如果被量化抹掉，
   把 `NCOLORS` 调大再试。
4. 把生成的 `<VAR>.dart` 拷进
   `lib/features/buddy/domain/pixel_arts/`，文件名用蛇形命名
   （如 `blue_guga.dart`）。

### 路线 B：纯手绘

直接手写 rows/palette（参考 `duck.dart` 等现有文件的画法）。要点：
底部对齐（脚贴到第 ~125 行附近）、左右居中、身体主色占大面积、
`E` 眼睛、可选 `p` 腮红。128×128 手写工作量大，建议先在 64×64 画好
再用 Scale2x 放大（见 git 提交 `c848d844` 里的放大脚本做法）。

## 2. 接入代码（共 3 处，全部必改）

### 2.1 枚举追加（`buddy_types.dart`）

在 `BuddySpecies` **末尾**追加（把原末尾的 `;` 改成 `,`）：

```dart
  nailong('奶龙'),
  blueGuga('蓝咕咕'),
  shyNailong('腼腆奶龙'),
  xinWuZhong('新物种中文名');   // ← 新增，enum 名用小驼峰
```

### 2.2 注册像素图（`pixel_arts/pixel_arts.dart`）

按字母序加 import，并在 `kBuddyPixelArts` 末尾注册：

```dart
import 'xin_wu_zhong.dart';
...
  BuddySpecies.shyNailong: shyNailongArt,
  BuddySpecies.xinWuZhong: xinWuZhongArt,
};
```

### 2.3 种子版本（`buddy_engine.dart` / `buddy_controller.dart`）

按 0.3 节的规则判断：当前 `v4-` 若未发布则什么都不用改；已发布则
冻结池子并升版本（照抄提交 `03132f8f` 的模式）。

UI 不用改：物种选择器、像素渲染、帽子叠加都自动遍历
`BuddySpecies.values` 和 `kBuddyPixelArts`。

## 3. 单测

`test/features/buddy/buddy_engine_test.dart` 里的格式校验（128×128、
palette 完整、有 `E`）和「v4 大样本覆盖全部物种」会自动覆盖新物种，
不用改。只有升种子版本时才需要新增一条「旧版本抽不到新物种」的
测试（照抄现有的 v2/旧种子测试）。

## 5. 常见坑清单

- [ ] 枚举插到了中间而不是末尾 → 所有老宠物变身，绝对禁止。
- [ ] 忘了在 `kBuddyPixelArts` 注册 → 渲染时 `art!` 空断言崩溃，
      单测「缺少像素图」会先拦住。
- [ ] 像素图没有 `E` → 单测失败，且宠物不会眨眼。
- [ ] `E`/`p` 被当普通颜色字符占用 → 眨眼/腮红渲染错乱。
- [ ] 行长或行数不是 128 → 单测失败。
- [ ] 已发布的种子版本池子没冻结就加物种 → 存量宠物变身。
- [ ] palette 里字符和 rows 对不上（多写/漏写）→ 单测
      「字符不在调色板里」失败。
